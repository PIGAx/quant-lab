TradingView Pine Script 產生器(v6)

產生的策略碼可直接貼進 TradingView Pine Editor 回測;內含 alert() 呼叫,建立警報時選「Order fills / alert() function calls」即可把 JSON 訊號推到 webhook(接 Telegram 見「⚙️ 同步與通知」)。

選好模板後按「產生 Pine 腳本」。

Pine 實戰備忘

主題重點
避免前視偏誤策略單預設下一根 K 開盤成交(process_orders_on_close=false),與本平台回測室、Python 腳本的 shift(1) 邏輯一致,三邊結果才可互相對照。
台股成本證交稅 0.3%(賣出)+ 手續費 0.1425%×2,來回約 0.585%;範本已用 commission_value 帶入。
警報 JSON範本的 alert() 會送出 {"ticker","action","price","time","strategy"},可被下方 Telegram 轉發服務直接解析。
免費方案限制免費版警報無 webhook,可改用「電子郵件→Gmail 篩選轉發」或直接採用本專案的 GitHub Actions 掃描(完全免費、不依賴 TV)。
Bar Replay 驗證寫完先用 Bar Replay 逐根確認訊號位置,再看回測績效,避免 repaint。指標中用到 request.security 時務必加 lookahead=barmerge.lookahead_off。

GitHub 跨裝置同步

把本頁部署到 GitHub Pages 後,策略資料會存到 repo 內 data/strategies.json,任何裝置開網頁都能拉取/推送,且每次變更都有 commit 紀錄可回滾。Token 只存在你目前這台裝置的瀏覽器,不會被寫進 repo。

Telegram 通知

① 在 Telegram 找 @BotFather → /newbot 取得 Bot Token ② 跟你的 bot 說一句話,開 api.telegram.org/bot<TOKEN>/getUpdates 找 chat id ③ 填入下方測試。日常訊號由 repo 內的 GitHub Actions 每日收盤後自動掃描發送(設定見 README)。

⚠ 測試按鈕需在 GitHub Pages 或本機開啟時才能連線(Claude 預覽環境會擋外部請求)。TradingView 付費版 webhook 亦可直接指向 Cloudflare Worker 轉發到 Telegram(Worker 範本在 repo 的 webhook/worker.js)。
